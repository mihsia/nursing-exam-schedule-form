# 護理師甄選複試時間分配表產生器

開啟 `app/index.html` 即可使用複試時間分配表。
開啟 `app/grouping.html` 可使用成績 S 型分組工具。

## 使用方式

1. 第一次使用先建立本機管理帳號與密碼；之後開啟表單需先登入。
2. 如有既有資料，先從歷史紀錄載入；第一次使用可直接往下填。
3. 填寫標題、複試日期、考場地點、組別與第一位報到時間。
4. 貼上准考證號碼，一行一筆，或用逗號、空白分隔；若尚未完成名冊，可在這一步開啟成績 S 型分組工具產出資料。
5. 依流程順序設定各階段與前一階段的分鐘差：
   - 報到到引導抽題
   - 引導抽題到實作抽題準備開始，並設定準備長度
   - 實作抽題準備結束到實作開始，並設定實作長度
   - 實作結束到口試預備教室
   - 口試預備到口試開始，並設定口試提醒與口試長度
6. 如需休息或午休，在休息規則中逐行設定。
7. 確認地點、頁尾備註與報表戳章後，儲存設定並列印或匯出 CSV。

休息規則格式：

```text
第幾位後:休息分鐘:顯示文字
```

休息會從各階段前一位結束後 1 分鐘開始，並顯示到下一位同階段開始前 1 分鐘；下一位時間會自動順延，避免休息列與下一位時間重疊。

範例：

```text
7:15:休息
14:55:午休
21:15:休息
```

## 輸出

- `列印 / 另存 PDF`：使用瀏覽器列印功能，可選擇另存成 PDF。
- `匯出 Word`：匯出 Office 365 Word 可開啟的 `.docx` 文件，表格內容可直接編輯，並保留目前報表戳章設定。
- `匯出 CSV`：匯出可用 Excel 開啟的時間分配表資料。
- `載入附件範例`：恢復成附件 A 組的範例資料與時間設定。
- `報表戳章`：可上傳甄選圓戳章，並設定是否顯示在報表中。

## 資料庫與歷史檔

- 表單會使用瀏覽器本機資料庫自動保存最後一次操作草稿。
- 可用 `儲存設定` 建立命名紀錄，之後從下拉選單載入。
- 主表單的自訂戳章與顯示設定會一起保存。
- 可用 `匯出歷史檔` 產生 JSON 檔，未來再用 `匯入歷史檔` 還原。
- 歷史檔保存在使用者電腦，不需要伺服器或資料庫主機。

## 登入與資安

- 在支援 PHP + MariaDB 的 NAS 環境中，登入會優先使用 `api/auth.php`，帳號密碼保存在 MariaDB，登入狀態由 PHP session 管理。
- 若部署在 GitHub Pages 等沒有 PHP 的靜態環境，會自動退回瀏覽器本機登入，密碼以加鹽 PBKDF2 雜湊保存在瀏覽器本機資料。
- 外網使用時仍建議搭配 HTTPS、NAS 防火牆、反向代理存取控制或 VPN/Tailscale，不建議只依賴前端登入。

### NAS MariaDB 登入設定

1. 在 Synology MariaDB 建立資料庫，例如 `nursing_exam`。
2. 建立專用資料庫使用者，例如 `nursing_exam_user`，只授權存取 `nursing_exam`。
3. 匯入 `app/api/schema.sql` 建立 `users` 資料表。
4. 在 NAS 的網站目錄中，將 `api/config.sample.php` 複製成 `api/config.php`。
5. 編輯 `api/config.php`，填入 MariaDB 主機、資料庫名稱、使用者與密碼。
6. 第一次開啟網站時，登入畫面會要求建立第一組管理帳號，之後兩個頁面共用同一個資料庫登入。

## 成績 S 型分組

1. 開啟 `app/grouping.html`。
2. 匯入含有 `准考證` 與 `成績` 欄位的 `.xlsx`、`.csv` 或 `.tsv`。
3. 輸入組數，最多 26 組。
4. 如只需部分考生進入複試，輸入 `預定進入複試人數`；留空則全部分組。
5. 點選 `產生 S 型分組`。
6. 使用 `匯出 CSV` 產出 `組別`、`准考證` 欄位資料。

分組方式：先依 `成績` 由高到低排序，取預定進入複試人數後，再依 A、B、C... 下一排反向 C、B、A... 的 S 型方式分配。
